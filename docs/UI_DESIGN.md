# D3keyHelper Linux UI 设计说明

本文档记录当前 GUI 的设计状态和后续维护准则。它不是下一版概念稿，而是和现有 `PySide6 + PySide6-Fluent-Widgets` 实现保持一致的设计说明。

![UI Preview](./ui-preview.svg)

## 设计结论

当前 GUI 定位是一个 **紧凑、低干扰、长期可维护的 Linux 桌面配置工具**。

已经落地的方向：

1. 保留 **PySide6 / Qt Widgets**，不切换 GTK、Qt Quick/QML 或 WebView。
2. 使用 **PySide6-Fluent-Widgets** 提供按钮、下拉框、输入框、表格和滚动区域的基础体验。
3. 通过项目内小型 QSS 统一字体、颜色、侧边栏、状态栏和技能表格。
4. 使用左侧导航、右侧设置页、顶部工具栏、底部状态栏、可展开日志的桌面工具布局。
5. 配置页的技能区域使用表格，避免把 6 个技能位做成大量重复表单。

不再采用旧文档中的 `qdarktheme` 或 `qt-material` 试验路线。当前依赖已经固定在 `PySide6-Fluent-Widgets>=1.11,<2`。

## 视觉目标

界面应保持这些特征：

1. **紧凑**：主要配置在 `1120 x 720` 默认窗口内可读，不依赖大量滚动。
2. **克制**：颜色只用于主操作、选中态、状态和错误提示。
3. **稳定**：启用/禁用、日志变化、语言切换不应造成明显布局跳动。
4. **清楚**：用户能快速区分全局配置、配置档配置、运行状态和最近日志。
5. **低疲劳**：浅色背景、清晰边界、适中行距，避免大面积装饰。

不要把界面做成营销页、网页后台或大卡片仪表盘。它应该更接近 KDE System Settings、GNOME Tweaks、VS Code Settings 这类桌面设置工具。

## 当前布局

主窗口采用四区结构：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部工具栏：语言 / 当前配置 / 重新载入 / 保存 / 启动 / 停止     │
├──────────────┬───────────────────────────────────────────────┤
│ 左侧导航      │ 当前设置页                                      │
│ 通用          │  页面标题 / 分组 / 表单 / 表格                    │
│ 配置1         │                                               │
│ 配置2         │                                               │
│ ...           │                                               │
│ 添加 / 删除   │                                               │
├──────────────┴───────────────────────────────────────────────┤
│ 底部状态栏：运行状态 / 最近日志 / 展开日志                      │
└──────────────────────────────────────────────────────────────┘
```

当前尺寸来自 `gui_widgets.py` 和 `d3keyhelper_linux_gui.py`：

1. 默认窗口：`1120 x 720`
2. 最小窗口：`960 x 620`
3. 顶部工具栏高度：`44px`
4. 左侧导航宽度：`150px`
5. 底部状态栏高度：`40px`
6. 内容区边距：左 `30px`、上 `18px`、右 `24px`
7. 日志默认收起，展开后高度约 `148px`

## 顶部工具栏

顶部工具栏只放高频动作：

1. 语言切换：`简 / EN / 繁`
2. 当前激活配置下拉框
3. 重新载入
4. 保存配置
5. 启动
6. 停止

规则：

1. `启动` 是唯一主按钮，使用 Fluent primary 样式和主色。
2. `停止` 不默认使用红色，避免把普通停止操作表现成危险操作。
3. 配置路径不放在工具栏里抢宽度；当前运行反馈交给底部状态栏和日志。
4. 工具栏不放说明文字，不放大标题。

## 左侧导航

左侧导航用于切换 `通用` 和所有配置档页面。

当前实现要点：

1. 宽度固定为 `150px`。
2. 使用 `ListWidget`，滚动只在配置档过多时出现。
3. 选中态为浅蓝背景和蓝色文字。
4. 支持多选配置档后删除，底部提供 `添加` 和 `删除` 两个操作。
5. 导航字体使用 CJK 友好的字体栈并保持 normal 字重。

维护规则：

1. 不要把导航做成大图标菜单。
2. 不要在导航项中加入长说明。
3. 添加/删除操作保持在侧边栏底部，避免和页面内容混在一起。

## 通用页

通用页是全局设置入口，当前为左右两列。

左列 `基础`：

1. 当前激活配置
2. 战斗宏启动方式和热键
3. 助手启动方式和热键
4. 发送模式
5. 游戏分辨率、Gamma、Buff 阈值
6. 宏启动瞬间执行一次、`d3only`、智能暂停、切换配置提示音

右列 `助手/输入`：

1. 自定义强制站立、强制移动、药水按键
2. 赌博、拾取、分解、重铸、升级、转化、丢装助手
3. 动画速度预设、辅助鼠标速度、辅助动画延迟
4. 安全格和最大重铸次数
5. Safezone 状态提示

维护规则：

1. 新增全局配置优先放入现有两列，不轻易新增第三列。
2. 高频启动和运行行为放左列，助手和输入细节放右列。
3. 长说明放 tooltip，页面上只保留必要的状态提示。

## 配置页

每个配置档页面分为三块：

1. 页面标题：配置档名称和简短副标题。
2. 配置档基础设置：基础、走位与药水、按键队列、快速暂停。
3. 技能表：6 个技能位的按键和策略。

配置档基础设置保持两列：

左列：

1. 基础
2. 走位与药水

右列：

1. 按键队列
2. 快速暂停

维护规则：

1. 配置档页面不要回到纵向大表单。
2. 运行逻辑相关设置应靠近：例如按键队列的开关和间隔必须在同一块里。
3. 禁用状态只改变可用性和颜色，不删除控件，不改变行位置。

## 技能表

技能配置使用表格是当前方向，避免 6 组重复表单挤占页面。

当前列：

1. 槽位
2. 按键
3. 策略
4. 间隔
5. 延迟
6. 随机
7. 优先级
8. 重复
9. 重复间隔
10. 触发

当前规格：

1. 行数固定为 6。
2. 表头高度 `30px`，行高 `32px`。
3. 横向滚动关闭，列宽按 `_SkillColumnDistributor` 随表格宽度比例分配。
4. 数字列紧凑，策略列更宽，随机列居中。
5. 只在需要时显示按键队列警告。

维护规则：

1. 新增技能字段前先确认是否真的需要进入主表格。
2. 不要让单个列名或英文翻译撑破表格。
3. 如果字段只对某种策略有效，保留位置并使用禁用态。

## 日志与状态

底部状态栏始终可见，日志默认收起。

状态栏显示：

1. 运行状态圆点
2. 运行器状态文字
3. 最近日志摘要
4. 展开/收起日志按钮

日志面板规则：

1. 默认隐藏，用户需要时展开。
2. 展开后显示 `PlainTextEdit`，最多保留 500 行。
3. 普通 runner event 不直接显示原始 `EVENT:` 行，而是转成用户可读日志。
4. 最近日志在状态栏使用 elide 处理，完整内容放 tooltip。

```text

警告/错误       #b42318
技能表背景      #f8faff

使用规则：

1. 蓝色用于主按钮、语言选中、导航选中和表格强调。
2. 绿色只用于运行中状态。
3. 红色只用于错误、危险和明确警告。
4. 不使用渐变、装饰光斑、强阴影或大面积品牌色背景。

DejaVu Sans

```

当前字号和字重：

1. 主体控件：`13px`，`font-weight: 400`
2. 工具栏标签：`12px`
3. 页面标题：`14px`
4. 日志：`12px`
5. 导航：`13px`，normal

维护规则：

1. 中英文界面必须使用同一套字号和字重策略。
2. 不使用 hero 大标题。
3. 英文标签需要验证是否被裁切。

## 控件规范

按钮：

1. 常规动作使用 Fluent `PushButton`。
2. 主动作使用 `PrimaryPushButton`。
3. 按钮配图标时优先使用 `FluentIcon`。

输入：

1. 文本输入使用 Fluent `LineEdit`。
2. 下拉使用 Fluent `ComboBox`。
3. 数字输入使用 Fluent `SpinBox` / `DoubleSpinBox`，不显示上下按钮。
4. 表单控件高度以 `26px` 为基础。

开关：

1. 布尔设置使用 `CheckBox`。
2. 带参数的开关保持同一行，例如“赌博助手 + 点击次数”。
3. 不把说明文字塞进 checkbox 文案。

提示：

1. 长说明放 tooltip。
2. 必须直接提醒的风险用 inline warning。
3. 不在主界面写使用教程。

## 交互规则

1. 配置变更自动保存。
2. 运行中修改关键配置时自动重启运行器，并在状态栏/日志中反馈。
3. 语言切换后重建界面，并保存到 `d3oldsand.ini`。
4. 添加配置后自动选中新配置。
5. 删除配置支持多选，但不能删除到 0 个配置。
6. 控件禁用时保留原位置，降低文字颜色。
7. 错误和警告要告诉用户当前状态，避免只显示失败。

## 代码位置

主要 UI 代码分布：

1. `d3keyhelper_linux_gui.py`：主窗口、工具栏、导航、状态栏、通用页、运行器进程。
2. `gui_widgets.py`：QSS、尺寸常量、表单/分组构建函数、tooltip、枚举项。
3. `gui_profile_page.py`：配置档页面和技能表。
4. `gui_i18n.py`：简体中文、繁体中文、英文切换。

修改 UI 时优先复用 `gui_widgets.py` 中的构建函数和尺寸常量，不要在页面代码里散落新的 magic number。

## 验收清单

每次较大 UI 改动后至少检查：

1. `1120 x 720` 默认窗口下主界面不显得空、不挤。
2. `960 x 620` 最小窗口下仍能操作。
3. 中文、英文、繁体中文三种语言都不裁切核心按钮和表格列。
4. 通用页两列内容对齐，Safezone 状态不挤占其他控件。
5. 配置页技能表无横向滚动，6 行完整可读。
6. 日志收起时不浪费空间，展开后不遮挡状态栏。
7. 启动、停止、运行器退出、配置变更重启都有状态反馈。
8. 禁用态清楚但不刺眼。
9. AppImage 环境下图标、字体、qfluentwidgets 控件正常显示。

## 后续方向

后续优化应围绕这几个方向做小步迭代：

1. 优先解决真实裁切、拥挤、状态不清楚的问题。
2. 保持浅色、紧凑、桌面工具风格。
3. 不引入新的主题库，除非能明确减少维护成本。
4. 不把当前设置工具改成品牌化或装饰化界面。
5. 截图和 `ui-preview.svg` 要跟随真实界面更新。
